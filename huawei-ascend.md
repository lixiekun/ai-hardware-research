## 核心结论

| 项目 | 结论 |
|------|------|
| **性能** | 910B3 单卡算力 313 TFLOPS，接近 A100 |
| **价格** | Atlas 800 (8卡) 中标价 **57.5万**，性价比高 |
| **GLM-5** | ✅ **8×910B 可跑 W4A8 量化版** (>45 tokens/s) |
| **建议** | 150万预算推荐 **方案一**：单台 Atlas 800 |

> ⚠️ 注意：GLM-5 BF16 完整版需 1.5TB 显存，但 **W4A8 量化版只需 ~300GB**，8×910B (512GB) 完全够用

## 一、芯片参数对比

### 1.1 昇腾910B vs 英伟达

> 来源：[知乎专栏 - NPU vs GPU参数速查](https://zhuanlan.zhihu.com/p/2004196636507789012)

| 参数 | **910B3** | A100 | A800 | H20 |
|------|:---------:|:----:|:----:|:---:|
| FP16 算力 | **313** TFLOPS | 312 | 312 | 148 |
| 显存容量 | 64GB | 80GB | 80GB | **96GB** |
| 显存带宽 | 1.6TB/s | 2TB/s | 2TB/s | **4TB/s** |
| 互联带宽 | 392GB/s | 600GB/s | 400GB/s | **900GB/s** |
| 功耗 | **310W** | 400W | 400W | 500W |

### 1.2 昇腾910C vs 英伟达高端卡

| 参数 | **910C1** | H100 | H200 | H20 |
|------|:---------:|:----:|:----:|:---:|
| FP16 算力 | 800 TFLOPS | **989** | **989** | 148 |
| FP8 算力 | ❌ | **1979** | **1979** | 296 |
| 显存容量 | **128GB** | 80GB | 141GB | 96GB |
| 显存带宽 | 3.6TB/s | 3.35TB/s | **4.8TB/s** | 4TB/s |

### 1.3 关键结论

```
┌─────────────────────────────────────────────────────────────┐
│  算力：910B ≈ A100 > H20                                    │
│  显存：H20(96GB) > A100(80GB) > 910B(64GB)                  │
│  推理性能：910B ≈ H100 的 60%                               │
│  训练能力：H100 > A100 > 910B > H20                         │
│  功耗效率：910B(310W) 优于 A100(400W)                        │
└─────────────────────────────────────────────────────────────┘
```

## 四、150万预算方案

### 方案一：单台 Atlas 800（推荐）

```
┌─────────────────────────────────────────────────────┐
│  配置：8× 昇腾910B3 (64GB)                           │
│  显存：512GB HBM2e                                  │
│  算力：2.24 PFLOPS                                  │
│  价格：80-120万 ✅ 预算内                            │
└─────────────────────────────────────────────────────┘
```

**适合场景：**
| 场景 | 支持度 |
|------|:------:|
| GLM-5 **W4A8 量化版** | ✅ **可运行** |
| GLM-4 系列训练+推理 | ✅ |
| DeepSeek 蒸馏版 | ✅ |
| MiniMax M2.5 | ⚠️ 需适配 |
| GLM-5 BF16 完整版 | ❌ 需 1.5TB |

#### GLM-5 W4A8 并发性能详解

**单请求性能：**
| 指标 | 数值 |
|------|------|
| 首 Token 延迟 | < 1.5 秒 |
| 解码速度 | **> 45 tokens/s** |
| 上下文 | 支持 200K |

**并发能力估算（假设平均对话 500 tokens）：**

| 并发用户 | 总吞吐 | 人均响应时间 |
|:--------:|:------:|:------------:|
| 5 人 | 225 tok/s | ~11秒 |
| 10 人 | 450 tok/s | ~11秒 |
| 15 人 | 675 tok/s | ~11秒 |
| 20 人 | 900 tok/s | ~11秒 |
| 25 人 | 1125 tok/s | ~11秒 |

**50-100 人在线场景：**

| 在线用户 | 同时请求* | 实际并发 | 体验 |
|:--------:|:---------:|:--------:|:----:|
| 50 人 | ~30% | 15 人 | ✅ 良好 |
| 75 人 | ~30% | 22 人 | ✅ 可接受 |
| 100 人 | ~30% | 30 人 | ⚠️ 可能排队 |

> *假设用户不会同时发请求，按 30% 并发率估算

**优化建议：**
| 方案 | 效果 | 来源 |
|------|------|------|
| 使用 SGLang | 吞吐 +15-35% | [华为云博客](https://bbs.huaweicloud.com/blogs/471427) |
| 批处理优化 | +20% | 调整 max-num-seqs |
| KV Cache 复用 | +10% | 长对话场景 |

```
┌─────────────────────────────────────────────────────┐
│  8×910B 跑 GLM-5 W4A8 并发结论：                    │
│  • 理论并发：15-25 人（无感排队）                    │
│  • 实际在线：50-80 人（30%并发率）                   │
│  • 极限负载：100 人（可能需要排队）                  │
│  • 优化后：可达 100+ 人                             │
└─────────────────────────────────────────────────────┘
```

**其他模型并发：**
- GLM-4-9B：50-80人
- GLM-4-32B：20-30人

## 六、风险与建议

### 风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 生态适配 | 部分模型需额外开发 | 优先选择已适配模型 |
| 价格波动 | 实际采购价可能差异大 | 多渠道询价 |
| 技术支持 | 生态资源相对较少 | 购买原厂服务 |
| 人才储备 | 熟悉昇腾的工程师较少 | 培训或外包 |

### 建议

1. **明确需求** - 确认主要使用的模型，优先选择已适配的
2. **多方询价** - 联系华为、宝德、神州数码等供应商
3. **POC测试** - 采购前申请测试机验证性能
4. **云上验证** - 先用华为云昇腾实例测试，再决定采购
5. **关注910C** - 如果预算允许，128GB显存更适合大模型
