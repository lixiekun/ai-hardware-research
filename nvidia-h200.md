## 三、模型适配

### 3.1 GLM-5.1 (744B 参数)

#### 显存需求与可行性

| 版本 | 显存需求 | H200 8卡 (1128GB) | H200 4卡 (564GB) |
|:----:|:--------:|:-----------------:|:----------------:|
| BF16 完整版 | ~1.5TB | ❌ | ❌ |
| **FP8 量化** | **~640GB** | ✅ **充裕** | ❌ |
| **INT4/FP8 混合** | **~440GB** | ✅ **轻松** | ✅ **可以** |
| W4A8 量化版 | ~300GB | ✅ | ✅ |
| 2-bit 极限量化 | ~241GB | ✅ | ✅ |

┌─────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────┤
│  显存占用  : ~440GB (1128GB 内非常充裕)             │
│  首 Token  : < 0.8 秒 (预估)                       │
│  解码速度  : > 80 tokens/s (预估)                   │

│  剩余显存  : ~688GB (可用于 KV Cache 和并发)        │

## 四、150万预算方案

### 方案一：H200 8卡服务器（推荐）

```
┌─────────────────────────────────────────────────────┐
│  配置：8× H200 SXM (141GB HBM3e)                    │
│  显存：1128GB HBM3e                                 │
│  算力：7.9 PFLOPS (FP16) / 15.8 PFLOPS (FP8)       │
│  价格：150-230万 ⚠️ 预算边界                         │
└─────────────────────────────────────────────────────┘
```

**适合场景：**
| 场景 | 支持度 |
|------|:------:|
| GLM-5.1 **FP8 量化版** | ✅ **最佳方案** |
| GLM-5.1 **INT4 量化版** | ✅ 充裕 |
| GLM-5 W4A8 量化版 | ✅ |
| MiniMax M2.5 FP16 | ✅ 仅需 2 卡 |
| DeepSeek-V3 4-bit | ✅ |
| GLM-5.1 BF16 完整版 | ❌ 需 1.5TB |

#### 并发性能估算

**GLM-5.1 INT4 量化版 @ 8×H200：**

| 在线用户 | 高峰并发 | 体验 |
|:--------:|:--------:|:----:|
| 80 人 | 24 人 | ✅ 流畅 |
| 100 人 | 30 人 | ✅ 良好 |
| 120 人 | 36 人 | ✅ 可接受 |
| 150 人 | 45 人 | ⚠️ 可能排队 |

> 基于 H200 比 910B 算力高 3.2 倍估算，实际需 POC 验证

## 六、风险与建议

### 风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 出口管制 | 审批延迟或限制 | 提前确认许可状态 |
| 价格波动 | 市场价 190-230万，超预算 | 多渠道询价，考虑租赁 |
| 供应周期 | 渠道交付可能较长 | 提前下单，锁定价格 |
| 制裁升级 | 后续可能限售 | 备选国产方案 |

### 建议

1. **预算 <150万** → 选昇腾 910B 或摩尔线程 S5000
2. **预算 150-230万** → H200 8卡是性能最优解
3. **先用后买** → 先租 H200 云实例验证，确认满足需求再采购
4. **混合部署** → H200 跑性能敏感模型 + 昇腾跑 GLM 原生模型
5. **关注 H200 对华审批进展** → 审批通过后价格可能更透明
